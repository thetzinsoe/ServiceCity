      - Implements `IDesignTimeDbContextFactory&lt;AppDbContext&gt;`
      - In `CreateDbContext(string[] args)`, reads the connection string from `ServiceCity/appsettings.json` at design time (fallback: hardcoded `Host=localhost;Port=5432;Database=servicecity_dev;Username=postgres;Password=REDACTED` if the file is not readable).
      - Builds an `NpgsqlDbContextOptionsBuilder` and returns a new `AppDbContext`.
      - Example structure:
        ```csharp
        using Microsoft.EntityFrameworkCore;
        using Microsoft.EntityFrameworkCore.Design;

        namespace ServiceCity.Data;

        public class DesignTimeDbContextFactory : IDesignTimeDbContextFactory&lt;AppDbContext&gt;
        {
            public AppDbContext CreateDbContext(string[] args)
            {
                var connectionString = "Host=localhost;Port=5432;Database=servicecity_dev;Username=postgres;Password=REDACTED";
                // Optionally attempt to read from ServiceCity/appsettings.json:
                // var configPath = Path.Combine(Directory.GetCurrentDirectory(), "../ServiceCity/appsettings.json");
                // if (File.Exists(configPath)) { ... parse JSON, extract ConnectionStrings.DefaultConnection ... }

                var optionsBuilder = new DbContextOptionsBuilder&lt;AppDbContext&gt;();
                optionsBuilder.UseNpgsql(connectionString);
                return new AppDbContext(optionsBuilder.Options);
            }
        }
        ```
    Purpose: The `IDesignTimeDbContextFactory` enables `dotnet ef migrations` to create an AppDbContext without a startup project. This avoids the circular dependency where Plan 01-01 Task 3 needs the Web project wired (ProjectReference + AddDbContext), but that wiring happens in Plan 01-02 (wave 2, depends_on: 01-01).
