# Plan 07-01: Customer Registration & Personalized Experience

## Objective

Enable customer registration, role-aware navigation, per-user booking ownership, admin customer management, and UI fixes (search bar layout, badge overlap).

Purpose: Transform ServiceCity from anonymous booking to identity-linked booking. Customers register once, sign in, and see only their own bookings. Admin gets a customer directory. Privacy hole (phone lookup) is closed.

Output: Working registration flow, role-aware home page, My Bookings page, Admin Customers page, inline search bar, fixed booking card badges, updated nav.

```
[Authorize]
public async Task<IActionResult> Customers(string? search)
{
    var query = db.Users.Where(u => !u.IsAdmin).AsQueryable();

    if (!string.IsNullOrWhiteSpace(search))
    {
        var term = search.Trim();
        query = query.Where(u =>
            u.Name.Contains(term) ||
            u.PhoneNumber.Contains(term) ||
            (u.PhoneNumberNormalized != null && u.PhoneNumberNormalized.Contains(term)));
    }

    var customers = await query
        .Select(u => new CustomerViewModel
        {
            Id = u.Id,
            Name = u.Name,
            PhoneNumber = u.PhoneNumber,
            BookingCount = db.Bookings.Count(b => b.UserId == u.Id),
            LastBooking = db.Bookings
                .Where(b => b.UserId == u.Id)
                .OrderByDescending(b => b.CreatedAt)
                .Select(b => (DateTime?)b.CreatedAt)
                .FirstOrDefault()
        })
        .OrderByDescending(c => c.BookingCount)
        .ThenBy(c => c.Name)
        .ToListAsync();

    return View(customers);
}
```

CustomerDetail GET:
```
[Authorize]
public async Task<IActionResult> CustomerDetail(int id)
{
    var user = await db.Users.FindAsync(id);
    if (user == null || user.IsAdmin) return NotFound();

    var bookings = await db.Bookings
        .Include(b => b.ServiceCategory)
        .Where(b => b.UserId == id)
        .OrderByDescending(b => b.CreatedAt)
        .ToListAsync();

    ViewBag.Customer = user;
    return View(bookings);
}
```

## Threat Model

| Boundary | Description |
|----------|-------------|
| Guest → Registration | Untrusted input (name, phone, address, password) crosses into User creation |
| Customer → Booking create | Authenticated user submits booking; UserId set from server-side claim |
| Admin → Customer list | Authenticated admin queries user data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | Registration form | mitigate | [ValidateAntiForgeryToken] on Register POST; ModelState validation; PasswordHasher for password storage |
| T-07-02 | Information Disclosure | Booking UserId | mitigate | UserId set from server-side ClaimTypes.NameIdentifier, never from client input |
| T-07-03 | Information Disclosure | Customer list | mitigate | [Authorize] on AdminController (existing); Customers/CustomerDetail actions require auth |
| T-07-04 | Elevation of Privilege | Registration | mitigate | Register action always sets IsAdmin=false; Admin created only via Setup action |
| T-07-05 | Spoofing | Phone uniqueness | mitigate | Check PhoneNumberNormalized uniqueness before creating user; prevent duplicate accounts |
| T-07-SC | Tampering | npm/pip/cargo installs | accept | No new packages added in this phase |
