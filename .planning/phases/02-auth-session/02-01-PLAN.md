<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Server (Setup form) | Untrusted input: Username, Password, ConfirmPassword, Name, PhoneNumber — all user-supplied strings |
| Server -> Database (EF Core) | User entity INSERT — password hash stored, never plaintext |
| libphonenumber -> Input | Phone number string parsed by third-party library — exception handling required |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Elevation of Privilege | AuthController.Setup GET/POST | mitigate | Both GET and POST check `!db.Users.AnyAsync(u => u.IsAdmin)` before allowing access. After first admin exists, returns 404. Per D-01/D-02/D-06. |
| T-02-02 | Elevation of Privilege | AuthController.Setup POST | mitigate | Setup page is one-time — after admin creation, the route returns 404 for all subsequent requests (GET and POST). No re-provisioning possible without direct DB manipulation. |
| T-02-03 | Information Disclosure | AuthController.Setup POST | mitigate | Password is hashed with PBKDF2 (PasswordHasher<T>, 100k iterations) before storage. PasswordHash stored in DB, never plaintext. Per D-09. |
| T-02-04 | Information Disclosure | Application logs/errors | mitigate | Never log Password or PasswordHash values. ModelState errors for validation failures use generic messages. Exception messages from libphonenumber are caught and replaced with user-friendly text (not exposed directly). |
| T-02-05 | Tampering | PhoneNumber input | mitigate | libphonenumber-csharp Parse() validates the number structure; IsValidNumber() checks it against Myanmar numbering rules. Server-side validation per D-12. Normalized to E.164 format before storage. |
| T-02-06 | Denial of Service | AuthController.Setup POST | accept | Single admin shop — setup form is a one-time operation. No brute-force vector on setup. Rate limiting for auth endpoints is scoped to Phase 6 (CROS-05). |
| T-02-07 | Tampering | CSRF on setup form | mitigate | Setup.cshtml includes `@Html.AntiForgeryToken()` in the POST form. ASP.NET Core MVC auto-validates antiforgery tokens on all POST requests by default. |
</threat_model>
