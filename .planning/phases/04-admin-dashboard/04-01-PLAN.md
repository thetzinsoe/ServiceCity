# Plan 04-01: Admin Dashboard

## Objective
Implement the admin dashboard for managing bookings, providing a grouped view by status and a detailed view for each booking.

## Tasks

### 1. Admin Controller Enhancements
**File:** `ServiceCity/Controllers/AdminController.cs`

- Add `Dashboard` (GET) action:
    - Retrieve all bookings, ordered by `CreatedAt` (newest first).
    - Group bookings by `Status` (Pending, Accepted, InProgress, Completed, Declined).
- Add `Details` (GET) action:
    - Receive `int id`.
    - Retrieve booking with included related data (User, ServiceCategory).
    - Return 404 if not found.

### 2. Admin Views
**File:** `ServiceCity/Views/Admin/Dashboard.cshtml`

- Display bookings grouped by status using sections.
- Use mobile-first design: stacking cards on narrow screens, grid/list layout on larger screens.
- Link each booking to its details page.

**File:** `ServiceCity/Views/Admin/Details.cshtml`

- Display comprehensive booking information (Customer details, Service Category, Address, Description, Date/Time, Status).
- Add "Back to Dashboard" link.

## Verification Checklist
- [ ] Admin can navigate to `/Admin/Dashboard`.
- [ ] Bookings are correctly grouped by status.
- [ ] Newest bookings appear first within each group.
- [ ] Admin can click a booking and view full details.
- [ ] Dashboard is mobile-responsive (stacked cards on narrow viewports).

## Files to Create/Modify

3. `ServiceCity/Views/Admin/Details.cshtml` (new)
