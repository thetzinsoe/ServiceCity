# Quick Plan: All Booking List with Status Filter

## What

Replace the "Check Status" nav link with an "All Bookings" link (authenticated users only) on the admin Dashboard page. Add a status filter dropdown (All / Pending / In Progress / Completed) above the booking cards so admins can quickly narrow the view. The existing search bar already handles customer name and phone search — no changes needed there.

## Why

The admin workflow currently requires scrolling past all status sections to find specific booking states. A status dropdown filter lets the admin focus on one status at a time. The "Check Status" link serves no purpose for the admin and clutters the nav.

## Tasks

### Task 1: Update Navigation — Remove "Check Status", Add "All Bookings"

**File:** `ServiceCity/Views/Shared/_Layout.cshtml`

1. Remove the "Check Status" nav item (lines 44-46):
   ```html
   <li class="nav-item">
       <a class="nav-link" asp-controller="Booking" asp-action="Lookup">Check Status</a>
   </li>
   ```
2. Change the authenticated "Bookings" nav link text to "All Bookings" (line 41):
   ```html
   <a class="nav-link" asp-controller="Admin" asp-action="Dashboard">All Bookings</a>
   ```

**Verify:** Load the homepage — "Check Status" is gone. Sign in as admin — "All Bookings" appears in the nav bar.

### Task 2: Add Status Filter to Admin Dashboard

**Files:**
- `ServiceCity/Models/AdminDashboardViewModel.cs`
- `ServiceCity/Controllers/AdminController.cs`
- `ServiceCity/Views/Admin/Dashboard.cshtml`
- `ServiceCity/wwwroot/css/site.css`

**Step 1 — ViewModel:** Add `string? StatusFilter` property to `AdminDashboardViewModel`.

**Step 2 — Controller:** In `Dashboard` action, add `string? status` parameter. When `status` is provided and valid, filter the query to only that `BookingStatus` before the existing search filter. Pass `status` into `model.StatusFilter`.

Controller logic:
```csharp
public async Task<IActionResult> Dashboard(string? search, string? status)
{
    var query = db.Bookings.Include(b => b.ServiceCategory).AsQueryable();

    // Status filter first
    if (!string.IsNullOrWhiteSpace(status) && Enum.TryParse<BookingStatus>(status, true, out var parsed))
    {
        query = query.Where(b => b.Status == parsed);
    }

    // Existing search filter
    if (!string.IsNullOrWhiteSpace(search)) { ... }

    // Build model with StatusFilter set
    var model = new AdminDashboardViewModel
    {
        Search = search,
        StatusFilter = status,
        ...
    };
}
```

**Step 3 — View:** Replace the static status section headers with a dropdown filter row above the search bar. The dropdown has options: All, Pending, In Progress, Completed, Declined. Clicking an option navigates to `?status=X` (preserving any existing `search` param). Show an active pill on the selected status. When a status is selected, show only that status's bookings (not all status sections).

Filter bar markup:
```html
<form asp-action="Dashboard" method="get" class="mb-3">
    <div class="d-flex flex-wrap gap-2">
        <a asp-action="Dashboard" asp-route-search="@Model.Search"
           class="btn btn-sm @(Model.StatusFilter == null ? "btn-primary" : "btn-outline-secondary")">All</a>
        <a asp-action="Dashboard" asp-route-status="Pending" asp-route-search="@Model.Search"
           class="btn btn-sm @(Model.StatusFilter == "Pending" ? "btn-primary" : "btn-outline-secondary")">Pending</a>
        <a asp-action="Dashboard" asp-route-status="InProgress" asp-route-search="@Model.Search"
           class="btn btn-sm @(Model.StatusFilter == "InProgress" ? "btn-primary" : "btn-outline-secondary")">In Progress</a>
        <a asp-action="Dashboard" asp-route-status="Completed" asp-route-search="@Model.Search"
           class="btn btn-sm @(Model.StatusFilter == "Completed" ? "btn-primary" : "btn-outline-secondary")">Completed</a>
        <a asp-action="Dashboard" asp-route-status="Declined" asp-route-search="@Model.Search"
           class="btn btn-sm @(Model.StatusFilter == "Declined" ? "btn-primary" : "btn-outline-secondary")">Declined</a>
    </div>
</form>
```

When `StatusFilter` is null, show all sections as today. When a status is selected, show only that status section.

**Step 4 — CSS:** Add a subtle pill-button style for the filter buttons in `site.css`:
```css
/* ── Status filter pills ── */
.status-filter .btn { border-radius: 20px; font-weight: 500; }
```

**Verify:** Navigate to `/Admin/Dashboard` — filter pills show above search bar. Click "Pending" — URL becomes `?status=Pending`, only Pending bookings appear. Click "All" — all statuses return. Combine with search: `?status=Pending&search=09` filters by both.
